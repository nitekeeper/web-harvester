How should custom naming work?

Three approaches for letting users set a custom name on a destination folder

A

Name prompt after picking

After the OS folder picker closes, an inline input appears pre-filled with the folder's real name. User edits or confirms, then it saves.

📁

↑ Appears inline in the list after folder is picked

B

Click-to-edit label (inline)

The destination label in the list is click-to-edit at any time. Clicking it turns it into an editable field in-place. Works for both new and existing destinations.

📁 ↵ to save

↑ User clicked the label — it became editable instantly

C

Edit (pencil) icon on each row

Each row in the list has a small pencil/edit icon. Clicking it opens an inline edit field. More explicit than click-to-edit; less automatic than the post-pick prompt.
