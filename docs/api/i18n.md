# i18n API

## Overview

The i18n module provides locale-aware message formatting for all user-facing strings. It loads
locale JSON bundles at runtime via Vite's `import.meta.glob`, formats ICU MessageFormat strings
via `@formatjs/intl-messageformat`, and maintains the active locale as module-level state.

English is always available as a fallback — missing keys in the active locale resolve to the
English entry; keys absent from English too return the message id verbatim.

Source: `src/domain/i18n/i18n.ts`

---

## Functions

### `loadLocale(locale: SupportedLocale): Promise<void>`

Activates the given locale. Loads the locale's `messages.json` bundle (and the English
fallback bundle on the first non-English call). State is mutated atomically — all async
operations complete before `currentLocale` or `activeMessages` change, so a mid-load failure
cannot leave the module in an inconsistent state.

```typescript
import { loadLocale } from '@domain/i18n/i18n';

await loadLocale('de');
```

Throws if the locale has no registered bundle (indicates an `_locales/` directory is missing
for a value in `SUPPORTED_LOCALES`).

---

### `formatMessage(id: string, values?: Record<string, string | number>): string`

Returns the formatted message string for `id` using the active locale. Substitutes `values`
into ICU MessageFormat placeholders. Falls back to the English entry when the key is absent
from the active locale. Returns `id` verbatim when the key is not found in either locale.

```typescript
import { formatMessage } from '@domain/i18n/i18n';

// Simple message
formatMessage('settings.title'); // → 'Settings'

// ICU variable substitution
formatMessage('notification.saved', { fileName: 'Note.md' });
// → 'Saved "Note.md"'

// ICU plural
formatMessage('highlights.count', { count: 3 });
// → '3 highlights'
```

Messages are defined in `src/domain/i18n/_locales/{locale}/messages.json` using the Chrome
extension message format: `{ "key": { "message": "ICU string" } }`.

---

### `getCurrentLocale(): SupportedLocale`

Returns the currently active locale code (default `'en'` before any `loadLocale` call).

---

### `isRTL(locale: SupportedLocale): boolean`

Returns `true` when `locale` requires right-to-left layout. Currently `'ar'`, `'fa'`, and
`'he'` are RTL. Use this to apply `dir="rtl"` to the document root.

```typescript
import { isRTL, getCurrentLocale } from '@domain/i18n/i18n';

document.documentElement.dir = isRTL(getCurrentLocale()) ? 'rtl' : 'ltr';
```

## Constants and Types

### `SUPPORTED_LOCALES`

Re-exported from `@shared/constants`. Tuple of 36 locale codes the extension ships with.

### `SupportedLocale`

Re-exported from `@shared/constants`. Union type of all supported locale code strings.

### `RTL_LOCALES: ReadonlySet<SupportedLocale>`

Pre-built `Set` of locales requiring RTL layout: `'ar'`, `'fa'`, `'he'`. O(1) membership
test — prefer this over a manual array check.

---

## Notes

- Locale bundles are discovered via `import.meta.glob('./_locales/*/messages.json')`. Vite
  includes all matching files in the production bundle as separate lazy chunks — no runtime
  HTTP fetch needed.
- The module state (`currentLocale`, `activeMessages`, `englishMessages`) is process-wide.
  In tests, reset between cases by calling `loadLocale('en')` in `beforeEach`.
- ICU syntax supports variables (`{name}`), plurals (`{count, plural, one {# item} other {# items}}`),
  and selects. See the [ICU MessageFormat spec](https://unicode-org.github.io/icu/userguide/format_parse/messages/) for the full grammar.
