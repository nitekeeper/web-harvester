# Application Services Catalog

Functions, classes, and types in `src/application/` and `src/core/`.

The design doc places both `application/` services and the `core/` micro-kernel in this catalog because `core/` is a peer layer (imports only from `shared/`) and is the foundation that application services build on.

---

## `src/core/` — Micro-Kernel

### `src/core/topology.ts`

Dependency graph utilities for the plugin registry.

| Name                      | Kind     | Description                                                                                                                                                                                                                                                                              |
| ------------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CircularDependencyError` | Class    | Thrown by `topologicalSort` when a cycle is detected. Message includes the cycle as `A → B → C`.                                                                                                                                                                                         |
| `topologicalSort(nodes)`  | Function | Sorts `DepNode[]` objects (each with `id` and `dependencies`) using Kahn's algorithm. Returns a `string[]` of IDs in activation order. Throws `CircularDependencyError` when the result is shorter than the input (cycle detected) and throws `Error` for missing dependency references. |

### `src/core/hooks.ts`

Four hook types and the concrete `CoreHookSystem` for inter-plugin communication. Each `tap()` is overloaded — the legacy `tap(id, handler, options?)` form (used inside `core/`) returns void, while the domain-facing `tap(fn)` form (used by plugins) returns an unsubscribe function. Hook payload data types (`ClipContent`, `HighlightEvent`, `Settings`, `ThemePreset`) live in `@domain/types` and are re-exported here for convenience.

| Name                            | Kind      | Description                                                                                                                                                                                                                                                                            |
| ------------------------------- | --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `TapOptions`                    | Interface | `{ priority?: number }` — registration options for waterfall and bail hooks (legacy form only).                                                                                                                                                                                        |
| `AsyncWaterfallHook<T>`         | Interface | Each handler can transform the passing value; handlers run in priority order (descending). Extends `IWaterfallHook<T>` from `@domain/types`. Methods: `tap(id, handler, options?)`, `tap(fn)`, `tapAsync(fn)`, `call(value)`, `untap(id)`.                                             |
| `createAsyncWaterfallHook<T>()` | Function  | Creates an `AsyncWaterfallHook`. Errors in handlers are caught and logged; subsequent handlers still run.                                                                                                                                                                              |
| `AsyncBailHook<T>`              | Interface | Any handler returning `false` short-circuits the chain. Methods: `tap(id, handler, options?)`, `call(value) → Promise<boolean>`, `untap(id)`.                                                                                                                                          |
| `createAsyncBailHook<T>()`      | Function  | Creates an `AsyncBailHook`. Errors are caught and logged; the chain continues unless `false` was returned before the error.                                                                                                                                                            |
| `AsyncEventHook<T>`             | Interface | Fire-and-forget async: all handlers run concurrently via `Promise.all`. Extends `IEventHook<T>` from `@domain/types`. Methods: `tap(id, handler, options?)`, `tap(fn)`, `tapAsync(fn)`, `call(value)`, `untap(id)`.                                                                    |
| `createAsyncEventHook<T>()`     | Function  | Creates an `AsyncEventHook`. Each handler is individually wrapped; one throwing does not prevent others.                                                                                                                                                                               |
| `SyncEventHook<T>`              | Interface | Synchronous fire-and-forget: handlers run sequentially in registration order. Extends `ISyncEventHook<T>` from `@domain/types`. Methods: `tap(id, handler)`, `tap(fn)`, `call(value)`, `untap(id)`. No priority.                                                                       |
| `createSyncEventHook<T>()`      | Function  | Creates a `SyncEventHook`. Errors caught per handler.                                                                                                                                                                                                                                  |
| `ClipResult`                    | Interface | Hook payload for `afterClip`: `{ fileName, destination }`.                                                                                                                                                                                                                             |
| `SaveRequest`                   | Interface | Hook payload for `beforeSave`: `{ content, fileName, destinationId }`.                                                                                                                                                                                                                 |
| `SaveResult`                    | Interface | Hook payload for `afterSave`: `{ filePath }`.                                                                                                                                                                                                                                          |
| `CoreHookSystem`                | Interface | Concrete hook map extending the domain `IHookSystem` with all implementation hooks: `beforeClip`, `onClip`, `afterClip`, `beforeSave`, `afterSave`, `onTemplateRender`, `onHighlight`, `onSettingsChanged`, `onThemeChanged`, `onPopupOpen`, `onPluginActivate`, `onPluginDeactivate`. |
| `createHookSystem()`            | Function  | Creates the full `CoreHookSystem` — instantiates all hook objects.                                                                                                                                                                                                                     |
| `ClipContent` (re-export)       | Interface | Re-exported from `@domain/types`. Hook payload for `beforeClip`/`onClip`: `{ title, url, body, selectedText, highlights? }`.                                                                                                                                                           |
| `HighlightEvent` (re-export)    | Interface | Re-exported from `@domain/types`. Hook payload for `onHighlight`: `{ id, url, text, xpath, color }`.                                                                                                                                                                                   |
| `Settings` (re-export)          | Type      | Re-exported from `@domain/types`. `Record<string, unknown>` — generic settings map for `onSettingsChanged`.                                                                                                                                                                            |
| `ThemePreset` (re-export)       | Interface | Re-exported from `@domain/types`. Hook payload for `onThemeChanged`: `{ id, name, base, tokens, isCustom }`.                                                                                                                                                                           |

### `src/core/registry.ts`

Plugin registry orchestrating lifecycle across all registered plugins.

| Name                                   | Kind      | Description                                                                                                                                                                                                                                                                                                                                                                    |
| -------------------------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `FailedPlugin`                         | Interface | `{ id: string, error: unknown }` — record of a plugin that failed to activate.                                                                                                                                                                                                                                                                                                 |

Inversify DI container factories.

### `src/core/ui-registry.ts`

In-memory implementation of the domain `IUIRegistry`. Plugins call this registry during `activate()` to contribute slot entries, named React components, and theme tokens; the host UI surfaces (popup, settings, side-panel) read from the same registry at render time.

| Name                 | Kind     | Description                                                                                                                                                                                                                                                                                |
| -------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `createUIRegistry()` | Function | Creates a fresh `IUIRegistry`. Backed by plain `Map`s for slot entries and named components, plus a single `Record<string, string>` for theme tokens. Slot entries default to "all contexts" (`popup`, `settings`, `side-panel`) when `addToSlot` is called without a `contexts` argument. |

### `src/core/types.ts`

Re-export barrel that gives plugin code access to the `TYPES` DI token map without importing from `@infrastructure/` directly (which would violate the layer rules enforced by `import/no-restricted-paths`). Plugins import `TYPES` from `@core/types`; only the composition root (`background.ts`) imports from `@infrastructure/adapters/tokens` directly.

| Name    | Kind   | Description                                                                                                    |
| ------- | ------ | -------------------------------------------------------------------------------------------------------------- |
| `TYPES` | Object | Re-exported from `@infrastructure/adapters/tokens`. Provides all nine adapter tokens plus five service tokens. |

### `src/application/ClipService.ts`

Application-layer orchestrator for the end-to-end clip flow: tab inspection, hook fan-out, persistence via an injected `saveTo`, and a user-facing notification on success. Stays within the application layer's import boundaries by accepting all infrastructure dependencies through locally-defined port interfaces. See `docs/api/clip-service.md` for the full public surface.

| Name                       | Kind      | Description                                                                                                                                                                                                                                                                                                                                       |
| -------------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Tab`                      | Interface | Lightweight `{ id, url, title }` tab shape — mirrors the infrastructure `Tab` so this service stays inside the application layer.                                                                                                                                                                                                                 |
| `ClipContent`              | Interface | `{ url, html, title }` — replicated from `@core/hooks` because `application/` cannot import from `core/`.                                                                                                                                                                                                                                         |
| `ClipResult`               | Interface | `{ fileName, destination }` — successful clip outcome returned to callers.                                                                                                                                                                                                                                                                        |
| `ClipAborted`              | Interface | `{ aborted: true, reason }` — value returned when `beforeSave` short-circuits the save.                                                                                                                                                                                                                                                           |
| `SaveRequest`              | Interface | `{ content, fileName, destinationId }` — payload passed to `beforeSave` taps.                                                                                                                                                                                                                                                                     |
| `SaveResult`               | Interface | `{ filePath }` — payload emitted to `afterSave` taps after a successful write.                                                                                                                                                                                                                                                                    |
| `Destination`              | Interface | `{ id, label, dirHandle, fileNamePattern, createdAt }` — mirrors the infrastructure `Destination` shape.                                                                                                                                                                                                                                          |
| `NotificationPayload`      | Interface | `{ type, title, message }` payload accepted by `INotificationAdapterPort`. Richer than the infrastructure adapter's bare-string surface so callers can express intent.                                                                                                                                                                            |

Mirrors the relevant slice of `ITabAdapter`.                                                                                                                                                                                                                                 |
| `IDestinationStoragePort`  | Interface | Minimal destination storage port — `getById(id)`. Mirrors the relevant slice of `IDestinationStorage`.                                                                                                                                                                                                                                            |
| `INotificationAdapterPort` | Interface | Notification port — `showNotification(payload: NotificationPayload)`. Defined locally because the application port shape differs from the infrastructure adapter.                                                                                                                                                                                 |
| `IClipHooksPort`           | Interface | Local port describing the four clip-flow hooks (`beforeClip`, `afterClip`, `beforeSave`, `afterSave`). Defined here because `application/` cannot import `@core/hooks` and the placeholder `IHookSystem` in `@domain/types` exposes hooks only as `unknown`.                                                                                      |
| `SaveToFn`                 | Type      | Function type matching `saveTo` from `@infrastructure/fsa` — injected via the constructor so the service stays inside the application layer's allowed import surface.                                                                                                                                                                             |

| `DestinationNotFoundError` | Class     | Thrown by `clip()` when `destinationId` does not match any persisted destination.                                                                                                                                                                                                                                                                 |

Constructor: `(tabAdapter, destinationStorage, hooks, notifications, saveTo, logger?)`. `clip()` reads the active tab, requests page HTML from the content script, runs `beforeClip`, resolves the destination, runs `beforeSave` (bails on `false`), persists, then fires `afterSave` + notification. |

### `src/application/SettingsService.ts`

Application-layer facade for the global app settings record. Wraps an `ISettingsStoragePort` with Zod validation, fans out in-process change notifications, and emits the `onSettingsChanged` plugin hook. See `docs/api/settings-service.md` for the full public surface.

| Name                   | Kind      | Description                                                                                                                                                                                                                                                                                                                                       |
| ---------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `AppSettingsSchema`    | Const     | Zod schema for the global settings record. Fields: `version`, `theme` (`light`/`dark`/`system`), `locale`, `defaultDestinationId?`, `conflictStrategy` (`overwrite`/`skip`/`suffix`), `customThemeTokens`, `customThemes[]`. All fields have defaults so an empty stored object parses successfully. Unknown keys are stripped on parse.          |
| `AppSettings`          | Type      | `z.infer<typeof AppSettingsSchema>` — the inferred TypeScript shape of the global settings record.                                                                                                                                                                                                                                                |
| `ISettingsStoragePort` | Interface | Minimal storage port required by `SettingsService` — `get(key)`, `set(key, value)`, `onChanged(handler)`. Mirrors the public surface of `createSettingsStorage` in `@infrastructure/storage` without depending on the concrete implementation.                                                                                                    |
| `ISettingsHooksPort`   | Interface | Local port describing the single hook this service emits — `onSettingsChanged.call(value)`. Defined in this file (rather than re-exporting from `@core/hooks`) because `application/` cannot import from `core/`.                                                                                                                                 |
| `ISettingsService`     | Interface | Public surface: `get()`, `set(key, value)`, `onChange(handler)`.                                                                                                                                                                                                                                                                                  |
| `SettingsService`      | Class     | Concrete implementation of `ISettingsService`. Constructor: `(storage: ISettingsStoragePort, hooks: ISettingsHooksPort, logger?: ILogger)`. `get()` returns schema defaults on missing/invalid data. `set(key, value)` performs a read-modify-write merge under the `app_settings` key, fires local subscribers, then awaits `onSettingsChanged`. |

| --------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `Highlight`           | Interface | `{ id, url, text, color, xpath, createdAt }` — a single text highlight on a web page. `id` is service-generated; `xpath` locates the highlighted node so it can be re-applied on revisit.                                                                                                  |
| `IStorageAdapterPort` | Interface | Minimal storage port — `getLocal(key)`, `setLocal(key, value)`, `removeLocal(key)`. Mirrors the relevant slice of `IStorageAdapter` from the infrastructure layer.                                                                                                                         |
| `IHighlightHooksPort` | Interface | Local port describing the single hook this service emits — `onHighlight.tap(name, fn)` / `onHighlight.call(highlight)`. Defined here because `application/` cannot import from `@core/hooks`.                                                                                              |
| `IHighlightService`   | Interface | Public surface: `addHighlight(url, text, xpath, color?)`, `getHighlightsForUrl(url)`, `removeHighlight(id)`, `clearHighlightsForUrl(url)`.                                                                                                                                                 |
| `HighlightService`    | Class     | Concrete implementation of `IHighlightService`. Constructor: `(storage: IStorageAdapterPort, hooks: Pick<IHighlightHooksPort, 'onHighlight'>, logger?: ILogger)`. `addHighlight` appends to the URL bucket, updates `highlight_index`, fires `onHighlight`, and returns the new highlight. |

### `src/application/ReaderService.ts`

Application-layer service that toggles a clean, distraction-free reader mode on a browser tab by injecting and removing the `READER_CSS` stylesheet from `@domain/reader/reader-styles`. Tracks active tabs in an in-memory `Set<number>` so a single `toggle(tabId)` method decides between `insertCSS` and `removeCSS`, and remembers the most recently toggled tab so `getState()` can report a snapshot. Stays inside the application layer's allowed import surface by accepting the tab adapter through a locally-defined `IReaderTabAdapterPort`. See `docs/api/reader-service.md` for the full public surface.

| Name                    | Kind      | Description                                                                                                                                                                                                                                                                                 |
| ----------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `IReaderTabAdapterPort` | Interface | Minimal tab port — `insertCSS(tabId, css)`, `removeCSS(tabId, css)`. Mirrors the relevant slice of `ITabAdapter` plus the `removeCSS` method needed to undo a previous injection.                                                                                                           |
| `ReaderState`           | Interface | `{ isActive, tabId }` — snapshot for the most recently toggled tab. `tabId` is `undefined` until `toggle()` has been called at least once.                                                                                                                                                  |
| `IReaderService`        | Interface | Public surface: `toggle(tabId: number): Promise<void>`, `isActive(tabId: number): boolean`, `getState(): ReaderState`.                                                                                                                                                                      |
| `ReaderService`         | Class     | Concrete implementation of `IReaderService`. Constructor: `(tabAdapter: IReaderTabAdapterPort, logger?: ILogger)`. `toggle()` flips the per-tab active state and calls `insertCSS` or `removeCSS` accordingly; `getState()` returns the snapshot for the last `tabId` passed to `toggle()`. |

### `src/domain/reader/reader-styles.ts`

| ------------ | ----- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `READER_CSS` | Const | Reader-mode stylesheet injected by `ReaderService.toggle()`. Defines a clean serif typography stack and hides common non-content elements (navigation, sidebars, ads) using `!important` so it overrides page styles. |

## `src/presentation/`

### `src/presentation/background/background.ts`

Composition root for the Chrome MV3 service worker. Wires concrete adapters, application services, the plugin registry, and the six plugins together at startup. See ADR-020 for the rationale behind the cross-layer imports allowed only for this file. Two sibling modules are extracted solely to keep `background.ts` under the 400-line limit and are not public APIs: `bridges.ts` (port-shape adapters between the domain compiler and application-layer services) and `wiring.ts` (lifecycle event wiring: `onInstalled`, context-menu, keyboard commands).

| Name                | Kind      | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `BackgroundContext` | Interface | Bag of objects produced by `bootstrap()`: `container`, `hooks`, `ui`, `registry`, `settingsStorage`, plus the five application services (`clipService`, `settingsService`, `templateService`, `highlightService`, `readerService`). Returned so callers (and tests) can inspect the wired graph.                                                                                                                                                                                                                                                  |
| `bootstrap()`       | Function  | Async composition root. Creates one `ChromeAdapter`, the root container, the hook system, the UI registry, both storage facades, the five application services (with bridge functions for the application/domain port shape mismatches), the plugin registry, and binds everything into the container. Registers and activates all six plugins, then wires `onInstalled`, the two top-level context-menu items (`clip-page`, `clip-selection`), and the keyboard command handler. Resolves with a `BackgroundContext` describing the wired graph. |

### `src/presentation/content/picker.ts`

Self-contained visual section picker mounted by the content script. Renders DOM overlays directly on the host page (no React, no application-layer imports) so it stays bundleable into a content script. The user hovers elements, clicks to toggle persistent exclude/include overlays, and walks the DOM tree with the bracket keys. `Enter` resolves with the chosen XPaths; `Escape` cancels.

| Name                   | Kind      | Description                                                                                                                                                                                                                                                                                   |
| ---------------------- | --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `PickerMode`           | Type      | `'exclude' \| 'include'` — which list of XPaths the user is selecting.                                                                                                                                                                                                                        |
| `PickerResult`         | Interface | `{ excludedXPaths?: string[]; includedXPaths?: string[] }` — result delivered to `onDone`. Only the field for the active mode is populated.                                                                                                                                                   |
| `PickerOptions`        | Interface | `{ mode: PickerMode; onDone: (result: PickerResult) => void; onCancel: () => void }` — picker callbacks.                                                                                                                                                                                      |
| `startPicker(options)` | Function  | Mounts the picker on the current document. Creates a hover overlay (`data-wh-overlay="hover"`), tracks `mousemove`/`click`/`keydown` (capture phase for mouse), and toggles persistent overlays per XPath. Returns a cleanup function (also called automatically before `onDone`/`onCancel`). |
| `computeXPath(el)`     | Function  | Builds a `/html/body/div[2]/p[1]`-style XPath for `el`. The `[n]` index is added only when same-tag siblings exist under the parent.                                                                                                                                                          |

### `src/presentation/content/content.ts`

Chrome MV3 content script entry point. Listens for picker control messages from the background service worker and mounts or dismounts `startPicker` accordingly. As a content-script entry point it is allowed to use the runtime onMessage API directly; the boundary ban is suppressed file-scoped.

| Name              | Kind | Description                                                                                                                                                                                                                                                                                |
| ----------------- | ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `IncomingMessage` | Type | Internal envelope `{ type: 'START_PICKER'; mode } \| { type: 'STOP_PICKER' }`. `START_PICKER` returns `true` from the listener so `sendResponse` may fire asynchronously after the user hits Enter or Escape; `STOP_PICKER` cleans up any active picker and replies with `PICKER_STOPPED`. |

### `src/presentation/stores/storageSyncMiddleware.ts`

Zustand middleware that mirrors a store's data slice into a browser-storage key via an injected `IStorageSyncPort`. The port is defined locally so the presentation layer does not import from `infrastructure/`. On every `set()` it strips function-typed properties from the new state and persists the rest; on creation it loads the existing record asynchronously and merges it; when `onChanged` fires for the same key it merges the external value into the store so popup / side panel / background stay in sync.

| Name                                               | Kind      | Description                                                                                                                                                                                                                                                                    |
| -------------------------------------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `IStorageSyncPort`                                 | Interface | Minimal storage port: `getLocal(key)`, `setLocal(key, value)`, `onChanged(handler)`. Mirrors the relevant slice of infrastructure's `IStorageAdapter`.                                                                                                                         |
| `stripFunctions<T>(state)`                         | Function  | Returns a copy of `state` with every function-valued property removed; used by the middleware so action methods never reach storage.                                                                                                                                           |
| `extractKeyChange<T>(changes, storageKey)`         | Function  | Pulls the `newValue` for `storageKey` out of an `onChanged` payload, narrowing it to `Partial<T>`. Returns `undefined` when the key is absent, the change record is missing, or the new value is not a plain object. Shared by `withStorageSync` and `bootstrapStore`.         |
| `withStorageSync<T>(storageKey, adapter)(creator)` | Function  | Curried Zustand middleware. First call captures `storageKey` and the adapter; the returned wrapper takes a `StateCreator<T, [], []>` and produces a new one that mirrors state to storage on every `set()`, hydrates from storage on creation, and reacts to external changes. |

### `src/presentation/stores/bootstrapStore.ts`

Generic async utility that wires chrome.storage sync to an existing Zustand singleton. Singletons in this codebase (`useSettingsStore`, `usePopupStore`) are created without `withStorageSync` so the entry point that owns the adapter (popup, settings, side-panel — see ADR-022) can plug in the concrete `IStorageSyncPort` at boot time. Loads the persisted slice first so the React tree never renders empty when storage already has data, then wires the persistence subscriber and the `onChanged` listener; an internal `applyingExternal` flag suppresses persistence while applying external changes so we never echo a value we just received.

| Name                                                      | Kind      | Description                                                                                                                                                                                                                                                                                                      |
| --------------------------------------------------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `BootstrapStoreOptions<T>`                                | Interface | Optional configuration: `serialize?: (state: T) => Partial<T>` selects the slice persisted to storage. Use to drop fields that are not serializable (e.g. `FileSystemDirectoryHandle` in the settings store). When omitted, every non-function property is persisted via `stripFunctions`.                       |
| `bootstrapStore<T>(adapter, storageKey, store, options?)` | Function  | Async — loads initial state, wires `store.subscribe()` for persistence, and wires `adapter.onChanged()` for inbound updates. Resolves once initial hydration completes (whether it succeeded or failed); subscribers are wired in either case so subsequent writes still flow even after a transient load error. |

### `src/presentation/settings/useDestinationHandlers.ts`

Hook that composes the FSA picker, the destination storage facade (read from `DestinationStorageContext`), and the global settings store into the three async handlers consumed by `DestinationsSection`. Each handler refreshes the `destinations` slice in the store after its mutation completes so the UI stays in sync with the IDB-backed truth. When no provider is mounted the handlers log a warning and return without mutating state, so render trees that do not need destination management stay safe to use.

| Name                       | Kind      | Description                                                                                                                                                                                                                              |
| -------------------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `DestinationHandlers`      | Interface | `{ onAdd; onRemove; onRename }` — all `Promise<void>`-returning. `onAdd` opens `window.showDirectoryPicker` and persists the choice; `onRemove` deletes by id; `onRename` updates the `label` field. Each refreshes the store afterward. |
| `useDestinationHandlers()` | Hook      | Returns memoised `DestinationHandlers` wired to the storage facade in context.                                                                                                                                                           |

### `src/presentation/settings/useTemplateHandlers.ts`

Hook that produces the three async handlers consumed by `TemplatesSection`. Templates live only in the Zustand settings store (no IDB), so each handler is a pure store mutation; persistence to chrome.storage is handled by `bootstrapStore` watching the settings store.

| Name                    | Kind      | Description                                                                                                                                                                            |
| ----------------------- | --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `TemplateHandlers`      | Interface | `{ onAdd; onRemove; onUpdate }` — all `Promise<void>`-returning. `onAdd` appends a new template with `crypto.randomUUID()` id; `onUpdate` merges field changes by id.                  |

### `src/presentation/side-panel/SidePanel.tsx`
