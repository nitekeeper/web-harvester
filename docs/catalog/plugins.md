# Plugins Catalog

Concrete plugins in `src/plugins/`. Each plugin implements `IPlugin` from `@domain/types`, owning its own activate/deactivate lifecycle and contributing UI, hooks, or services to the host.

---

## `src/plugins/theme/ThemePlugin.ts`

Manages the active theme: applies CSS custom-property tokens, watches the system `prefers-color-scheme` media query, and listens to the `onThemeChanged` hook so plugins can switch themes at runtime. Contributes a `settings-section` slot referencing the `ThemeSettingsPanel` component (resolved lazily by name).

| Name          | Kind  | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ------------- | ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ThemePlugin` | Class | Concrete `IPlugin`. Manifest: `{ id: 'theme', name: 'Theme', version: '1.0.0' }`. `activate()` reads the persisted `theme` key (a `ThemePreset`), applies its `base` (toggling `dark`/`light` on `document.documentElement`) and any `tokens` as CSS custom properties; falls back to `prefers-color-scheme: dark` when storage is empty. Subscribes to `onThemeChanged` and the system media query. `deactivate()` removes the media query listener and unsubscribes from the hook. |

## `src/plugins/clipper/ClipperPlugin.ts`

The root clip-action plugin. Validates that `ITabAdapter`, `INotificationAdapter`, and `IClipService` are bound in the container at activation time (failing fast rather than at user action), then registers `SaveButton` in the `popup-footer` slot. All other plugins that participate in the clip flow declare `dependencies: ['clipper']` so they are guaranteed to activate after this one.

| Name            | Kind  | Description                                                                                                                                                                                                                                                                                                                                                                                                      |
| --------------- | ----- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ClipperPlugin` | Class | Concrete `IPlugin`. Manifest: `{ id: 'clipper', name: 'Clipper', version: '1.0.0', requiredAdapters: [TYPES.ITabAdapter, TYPES.INotificationAdapter] }`. `activate()` eagerly resolves `ITabAdapter`, `INotificationAdapter`, and `IClipService` from the container (throws if any are missing), then registers `{ component: 'SaveButton', order: 100 }` in the `popup-footer` slot. `deactivate()` is a no-op. |

## `src/plugins/template/TemplatePlugin.ts`

Wires the user's active clip template into the `beforeClip` waterfall: calls `getDefault()` then `render(template.id, content)` and, on success, replaces `content.body` with the rendered output. Also registers a pass-through tap on `onTemplateRender` for downstream plugins to transform rendered output. Contributes `TemplateSelector` to `popup-properties` and `TemplateSettingsPanel` to `settings-section`. Declares `dependencies: ['clipper']`.

| Name             | Kind  | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ---------------- | ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `TemplatePlugin` | Class | Concrete `IPlugin`. Manifest: `{ id: 'template', name: 'Template', version: '1.0.0', dependencies: ['clipper'] }`. `activate()` resolves `ITemplateService` from the container, registers `TemplateSelector` (order `100`) in `popup-properties` and `TemplateSettingsPanel` (order `20`) in `settings-section`, taps `hooks.beforeClip` (`tapAsync`): `getDefault()` → `render(id, content)` → replaces `content.body` on `result.ok`, passes unchanged otherwise; taps `hooks.onTemplateRender` as a pass-through. Stores both unsubscribe handles. `deactivate()` invokes both and nulls them. |

## `src/plugins/highlighter/HighlighterPlugin.ts`

Bridges `IHighlightService` (application layer) into the plugin hook system. On the `beforeClip` waterfall it injects persisted highlights for the clipped URL into `content.highlights` (as a newline-joined string of highlight texts) when any exist, otherwise passes content through unchanged. On the `onHighlight` event hook it persists incoming `HighlightEvent`s via `addHighlight()`. Contributes a `popup-toolbar` slot referencing the `HighlighterButton` component. Depends on `clipper`.

| Name                | Kind  | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------- | ----- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `HighlighterPlugin` | Class | Concrete `IPlugin`. Manifest: `{ id: 'highlighter', name: 'Highlighter', version: '1.0.0', dependencies: ['clipper'] }`. `activate()` resolves `IHighlightService` and `ITabAdapter` from the container, registers `HighlighterButton` in `popup-toolbar`, and taps `hooks.beforeClip` (read-on-call: `getHighlightsForUrl(content.url)` → injects joined texts into `content.highlights` when non-empty) and `hooks.onHighlight` (`addHighlight(event.url, event.text, event.xpath, event.color)`). `deactivate()` invokes both unsubscribe handles and resolves cleanly even if called before activate. |

## `src/plugins/reader/ReaderPlugin.ts`

Contributes a reader-mode toggle button to the popup toolbar that delegates to `IReaderService` (resolved from the DI container at activation time). The plugin holds no internal state across `activate()`/`deactivate()` and taps no hooks — it is a pure UI surface registration. The local `IReaderServicePort` interface declares `toggle(tabId?: number)` so the toolbar `onClick` (which has no tab context at click time) can call `toggle()` without arguments while remaining structurally compatible with the concrete service.

| Name           | Kind  | Description                                                                                                                                                                                                                                                                                                                                                                                                     |
| -------------- | ----- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ReaderPlugin` | Class | Concrete `IPlugin`. Manifest: `{ id: 'reader', name: 'Reader', version: '1.0.0' }`. `activate()` resolves `IReaderService` from the container and registers `{ component: 'ReaderButton', order: 200, onClick: () => readerService.toggle() }` in `popup-toolbar`, then logs activation. `deactivate()` is a no-op (service lifecycle is managed by the DI container). No plugin dependencies; no hooks tapped. |

## `src/plugins/settings/SettingsPlugin.ts`

Contributes the general settings UI surface (a navigation entry plus a settings panel) and a debug panel, then persists incoming settings changes by tapping the `onSettingsChanged` event hook and forwarding the payload to `ISettingsService`. The plugin defines a local `ISettingsServicePort` (`set(settings: Settings): Promise<void>`) so the hook payload — a complete `Settings` record — can be forwarded as-is without depending on the application-layer key/value `set<K extends keyof AppSettings>` signature (see ADR-005 on plugin-local ports).

| Name             | Kind  | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ---------------- | ----- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `SettingsPlugin` | Class | Concrete `IPlugin`. Manifest: `{ id: 'settings', name: 'Settings', version: '1.0.0' }`. `activate()` resolves `ISettingsService` and `IPluginRegistry` from the container, registers `GeneralSettingsNav` (order `0`) in the `settings-nav` slot, registers `GeneralSettingsPanel` (order `0`) and `DebugPanel` (order `9999`, sorted to the end) in the `settings-section` slot, then taps `hooks.onSettingsChanged` via `tapAsync` to forward the new settings record to `settingsService.set(settings)`. Stores the unsubscribe handle for cleanup. `deactivate()` invokes the unsubscribe and resolves cleanly even if called before `activate()`. No plugin dependencies declared. |
